About the Complex Function Plotter

The complex function plotter plots a function \(f:\mathbb{C}\rightarrow \mathbb{C}\), by taking each pixel as a complex number. Roughly, for a pixel at position (x, y), it will calculate \( f(\mathrm{scale}(x)+\mathrm{scale}(y)i) \), where 'scale' represents the scaling and offset depending on the window size and position.
It then colors the pixel based on the value of \(f\) at that point; the hue varies with the argument of the value, and the lightness varies with the modulus of the value, so that zero is black and infinity is white.

You can use pretty much any alphanumeric string as a variable, except the reserved numbers (\(i, e,\) and \(pi\)) and function names. Well, with the caveat that you might sometimes be able to use a reserved function name as a variable, depending on its position in the expression (see "sin sin")

Supported Functions (along with the usual +, -, *, /, and ^)

Function Meaning
sin, cos, tan Yer standard trig functions, \( \cos(z) = \dfrac{e^{i z} + e^{-i z}}{2} \) and so on
exp \( \exp(z) = e^{z} \)
ln, log Log base \(e\)
log[n] Log base \(n\) — e.g. "log2(z)"
sinh, cosh Hyperbolic sin and cos
tet \( \mathrm{tet}(z, n) = \left. z^{z^{...^z}} \right\} n \) copies of \(z\).
If \(n\) is not an integer, it will be turned into one via \( n\mapsto \mathrm{floor}(\mathrm{Re}(n)) \)
Re, Im The real/imaginary part of the input

If hovering over the input boxes doesn't work, the inputs are from left to right: the function, the width, and the center. Clicking the button with the lock will add an input for height, after the width input.

Non-obvious buttons:
